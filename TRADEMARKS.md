# Trademark and Project Name Usage Policy

Status: public project-name usage boundary  
This document is not legal advice.

## Purpose

This document explains how the AAEF-AI-VA project name, repository name, slogans,
and related identifiers should be used.

It helps prevent confusion between:

- the original AAEF-AI-VA project,
- forks or modified versions,
- commercial services,
- customer-specific integrations,
- unrelated products.

## No registered trademark claim

Unless separately registered and documented, this project does not claim that
AAEF-AI-VA is a registered trademark.

This document records project-name and source-identification expectations, not a
registered trademark status claim.

## Project identifiers

Project identifiers include:

- `AAEF-AI-VA`
- `AAEF-controlled AI Vulnerability Assessment`
- `AI output is not authority`
- `Model output is not authority`
- repository URL: https://github.com/mkz0010/AAEF-AI-VA

## Permitted descriptive use

You may use the project name descriptively to refer to the original public project,
for example:

- "built using AAEF-AI-VA"
- "based on AAEF-AI-VA"
- "forked from AAEF-AI-VA"
- "compatible with concepts from AAEF-AI-VA"

Such use must not imply endorsement, certification, commercial partnership, audit
approval, legal approval, or production readiness.

## Use that requires separate permission

Separate written permission is required before using the project name or identifiers
to suggest:

- official endorsement,
- official commercial partnership,
- certified compatibility,
- managed service authorization,
- customer-ready production readiness,
- legal approval,
- audit opinion,
- compliance certification,
- permission to scan third-party systems.

## Forks and modified versions

Forks and modified versions should clearly identify themselves as forks or modified
versions.

They should not present themselves as the original AAEF-AI-VA project unless they
are maintained by the project author or an authorized maintainer.

## Commercial services

Commercial services using, adapting, or referencing AAEF-AI-VA should avoid customer
confusion.

Commercial service names, marketing pages, proposals, and customer-facing materials
should clearly distinguish:

- the public AGPL-3.0 reference implementation,
- any separate commercial offering,
- any customer-specific deployment,
- any modified or proprietary component.

## Prohibited implications

Do not imply that AAEF-AI-VA grants:

- runtime execution authorization,
- scan authorization,
- credential injection permission,
- customer-target operation permission,
- compliance certification,
- legal approval,
- audit opinion,
- safety guarantee,
- replacement for professional security judgment.

## No legal advice

This document is a project policy and communication boundary. It is not legal
advice.
