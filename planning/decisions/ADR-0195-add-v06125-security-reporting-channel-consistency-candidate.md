# ADR-0195: Add v0.6.125 SECURITY.md reporting-channel consistency candidate

Status: Accepted
Date: 2026-05-10

## Context

v0.6.124 selected SECURITY.md reporting-channel consistency as the next Medium-risk work item and assigned two checkpoints.

## Decision

Add a SECURITY.md reporting-channel consistency candidate.

The candidate clarifies the private reporting channel for sensitive reports, limits public GitHub issues to non-sensitive coordination, keeps repository security reports scoped to AAEF-AI-VA repository concerns, separates commercial/NDA discussions from vulnerability reporting, and reiterates that this repository is not authorization to test third-party systems.

## Consequences

v0.6.126 should review and decide whether the candidate wording is complete.

No runtime behavior, validator behavior, schema, public sample, AAEF main handback state, or external submission state is changed.
