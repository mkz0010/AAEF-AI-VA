# ADR-0073: Add v0.6.2 evaluation criteria and acceptance model

Status: Accepted  
Date: 2026-05-03

## Context

AAEF-AI-VA v0.6.0 established work ordering and v0.6.1 inventoried capabilities and
maturity.

The next step is to define how future capability improvements should be evaluated
before choosing whether to build a local lab, create demos, harden runtime gates, or
prepare commercial PoC boundaries.

## Decision

Add a v0.6.2 evaluation criteria and acceptance model.

The checkpoint records:

- evaluation principle,
- acceptance model summary,
- evaluation dimensions,
- maturity advancement rules,
- required acceptance gates,
- failure criteria,
- evidence requirements,
- evaluation record expectations,
- local lab decision criteria,
- demo-readiness criteria,
- commercial PoC boundary criteria,
- scoring guidance,
- runtime and scanning prohibitions.

## Consequences

Positive:

- Future implementation work has clearer acceptance criteria.
- Local lab and demo work can be judged before it is built.
- Maturity advancement is separated from unsupported readiness claims.
- Safety and claim boundaries remain part of evaluation.

Negative / deferred:

- This does not build the local lab.
- This does not enable runtime execution.
- This does not create a scoring certification.
- This does not demonstrate production deployment.
