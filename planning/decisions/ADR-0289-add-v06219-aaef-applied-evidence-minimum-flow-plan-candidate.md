# ADR-0289: Add v0.6.219 AAEF Applied Evidence Minimum Flow Plan Candidate

Status: Candidate recorded

## Context

v0.6.218 recorded AAEF main feedback and selected `aaef_applied_evidence_minimum_flow_plan` as the next high-risk work item.

AAEF main emphasized that AAEF-AI-VA should first reorganize existing Tool Gateway runner, mock flow, evidence chain linkage, reconstruction report, and reviewer materials into a minimum applied evidence package before adding new diagnostic capability.

## Decision

Record v0.6.219 as the AAEF Applied Evidence Minimum Flow Plan Candidate.

The candidate plan covers:

- central proposition
- minimum flow
- existing element inventory
- four-scenario matrix
- evidence linkage table
- tool_action_request record planning
- gate_decision record planning
- dispatch / non-dispatch evidence planning
- evidence event package planning
- reviewer walkthrough planning
- AAEF five questions mapping
- evidence trust boundary and non-proof boundary
- handback summary structure
- public/private/patent-sensitive boundary
- future sequencing

## Review boundary

The candidate plan is not accepted by this ADR.

v0.6.220 must review the candidate plan and record whether it is accepted, revised, or rejected.

## Consequences

The repository gains a reviewable plan candidate without creating minimum flow artifacts.

No fixtures are created or modified.

No Tool Gateway behavior, adapter behavior, runtime behavior, README, public cleanup, or publication status is changed.

Runtime demo remains necessary but deferred.

Publication remains deferred.
