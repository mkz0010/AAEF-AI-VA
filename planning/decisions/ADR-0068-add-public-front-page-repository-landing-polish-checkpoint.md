# ADR-0068: Add public front page and repository landing polish checkpoint

Status: Accepted  
Date: 2026-05-03

## Context

AAEF-AI-VA now has public trust and reviewer navigation, but the repository front
page also needs an explicit landing-page standard.

The repository may be discovered before any direct sales meeting. It should
therefore act as a conservative product introduction and trust surface without
becoming a sales deck or making unsupported claims.

## Decision

Add a v0.5.8 public front page and repository landing polish checkpoint.

The checkpoint records:

- first-minute reviewer goals,
- above-the-fold README expectations,
- public value proposition,
- product introduction boundary,
- reviewer landing paths,
- trust-signal placement,
- commercial-use placement,
- security placement,
- validation placement,
- README compatibility relationship,
- public trust reviewer navigation relationship,
- runtime and scanning prohibitions.

## Consequences

Positive:

- First-time reviewers get a clearer front-page experience.
- GitHub can function as a public product introduction surface.
- Trust signals become easier to find.
- Commercial-use boundaries remain visible without creating a sales deck.
- Unsupported production, certification, legal, audit, and scan-authorization claims remain avoided.

Negative / deferred:

- This does not rewrite the full README.
- This does not create commercial sales material.
- This does not publish pricing.
- This does not authorize runtime execution.
