# ADR-0196: Add v0.6.126 SECURITY.md reporting-channel consistency review and decision

Status: Accepted
Date: 2026-05-10

## Context

v0.6.125 prepared SECURITY.md reporting-channel consistency as checkpoint 1 of 2 for the Medium-risk public-facing documentation work item selected in v0.6.124.

## Decision

Review and accept the SECURITY.md reporting-channel consistency candidate.

The work item is closed as documentation-only. It clarifies private reporting for sensitive reports, limits public GitHub issues to non-sensitive coordination, keeps repository security reports scoped to AAEF-AI-VA repository concerns, separates commercial/NDA discussions from vulnerability reporting, and reiterates that this repository is not authorization to test third-party systems.

## Consequences

Future work should select the next item using the v0.6.120 risk-tiered checkpoint granularity policy.

No runtime behavior, validator behavior, schema, public sample, AAEF main handback state, or external submission state is changed.
