# ADR-0292: Add v0.6.222 Existing Element Inventory Candidate

Status: Candidate recorded

## Context

v0.6.221 selected `existing_element_inventory` as the next high-risk work item under the accepted AAEF Applied Evidence Minimum Flow Plan.

The accepted plan requires inventorying existing Tool Gateway runner, mock flow, evidence chain, reconstruction, runtime scaffold, safe demo, and reviewer materials before creating new minimum flow package artifacts.

## Decision

Record v0.6.222 as the Existing Element Inventory Candidate.

The candidate inventory classifies existing elements by:

- minimum flow step
- source location
- source class
- public/private/patent-sensitive boundary
- current coverage
- maturity boundary
- follow-up need

## Review boundary

The candidate inventory is not accepted by this ADR.

v0.6.223 must review the candidate inventory and record whether it is accepted, revised, or rejected.

## Consequences

The repository gains a reviewable existing element inventory candidate without creating a minimum flow package.

No fixtures are created or modified.

No evidence linkage, request, decision, dispatch, walkthrough, mapping, or handback records are created.

Runtime demo remains necessary but deferred.

Publication remains deferred.
