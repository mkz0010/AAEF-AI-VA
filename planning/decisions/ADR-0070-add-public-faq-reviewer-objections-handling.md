# ADR-0070: Add public FAQ and reviewer objections handling

Status: Accepted  
Date: 2026-05-03

## Context

AAEF-AI-VA now has public front page polish, reviewer navigation, and an evidence
and capability boundary summary.

The next public-trust need is to answer likely first-time reviewer objections
directly. The repository may be evaluated without a sales call, so common questions
about scanners, runtime execution, customer targets, AGPL-3.0, commercial licensing,
security reporting, and evidence-backed claims should be answered in public.

## Decision

Add a v0.5.10 public FAQ and reviewer objections handling checkpoint.

The checkpoint records:

- short answer,
- FAQ answers,
- reviewer objections and direct responses,
- evidence-backed answer pattern,
- relationships to evidence summary, front page polish, and public trust navigation,
- runtime and scanning prohibitions.

## Consequences

Positive:

- The public repository can answer likely objections before direct contact.
- Commercial and technical reviewers get clearer boundaries.
- Public claims remain evidence-backed and conservative.
- Runtime, scanning, credential, customer-target, legal, compliance, and audit boundaries remain explicit.

Negative / deferred:

- This does not create private sales material.
- This does not publish pricing.
- This does not create a commercial contract.
- This does not authorize runtime execution.
