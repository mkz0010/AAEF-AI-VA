# ADR-0148: Add v0.6.78 public sample relationship-to-validator planning

Status: Accepted  
Date: 2026-05-07

## Context

v0.6.76 selected public sample relationship to validator as the next Applied Evidence gap to plan. Tag v0.6.77 was used for repository hygiene cleanup after v0.6.76, so this substantive checkpoint uses v0.6.78.

Before any public sample or validator changes are considered, the project should define how a future documentation-only candidate can explain the relationship between the public sample and public validator.

## Decision

Add v0.6.78 as public sample relationship-to-validator planning.

This checkpoint plans a future documentation-only relationship candidate and preserves the current public sample, validator behavior, validator output, fixture metadata, and runtime boundaries.

It does not modify validator behavior, add validator output, create an output contract, add metadata fields, rewrite fixtures, add JSON Schema, add fixtures, start Applied Evidence implementation work, generate packages, create private review records, promote public samples, refine public sample content, change public example text, create a new walkthrough, prepare AAEF handback material, or authorize runtime/scanner/Docker/credential/customer/delivery activity.

## Consequences

The next checkpoint may create a narrow documentation-only public sample relationship-to-validator candidate if it preserves the planning boundaries and avoids public sample, validator, fixture, handback, and runtime changes.
