# ADR-0149: Add v0.6.79 public sample relationship-to-validator candidate

Status: Accepted  
Date: 2026-05-07

## Context

v0.6.78 planned a documentation-only public sample relationship-to-validator aid and preserved public sample, validator, fixture, handback, and runtime boundaries.

The project can now add a narrow relationship candidate that explains how reviewers should understand the retained public sample, retained public validator, and retained public negative fixtures without changing them.

## Decision

Add v0.6.79 as a public sample relationship-to-validator candidate.

This checkpoint creates the relationship candidate inside the v0.6.79 document and keeps it documentation-only.

It does not modify validator behavior, add validator output, create an output contract, add metadata fields, rewrite fixtures, add JSON Schema, add fixtures, start Applied Evidence implementation work, generate packages, create private review records, promote public samples, refine public sample content, change public example text, create a new walkthrough, prepare AAEF handback material, or authorize runtime/scanner/Docker/credential/customer/delivery activity.

## Consequences

The next checkpoint should review the relationship candidate and decide whether it is close-ready before any further Applied Evidence gap work is considered.
