## Boundary rationale

The public repository should not expose draft commercial packaging or pricing material before the project has an explicit commercial publication decision. Keeping those files in the public tree can confuse external readers because the README preserves a boundary that this is not a customer-ready managed assessment platform.

This cleanup only removes the files from the current public tree. It does not claim that previously published Git history is erased.
